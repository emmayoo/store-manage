# YOOPS 개인 중심 근무·매장 관리 서비스 기획안 v1

> 역할 가정: **10년차 시니어 서비스 기획자 관점**  
> 목적: Cursor / 개발자 / 디자이너가 그대로 이해하고 구현 가능한 수준의 **제품 정의 + 구조 명세**

---

## 1. 서비스 개요

### 1-1. 서비스 한 줄 정의

**YOOPS는 여러 매장에 소속된 개인 사용자가 자신의 근무, 일정, 매장 공지, 업무를 한 번에 관리할 수 있는 통합 스케줄·운영 도구다.**

> 기존 엑셀/카톡/종이 스케줄의 단절을 해결하고,
> “내 기준”으로 매장과 근무를 바라보게 만드는 것이 핵심 가치

---

## 2. 핵심 타겟

### Primary User (개인 사용자 중심)
- 편의점 / 소매점 / 소규모 매장에서 **직접 근무하는 사람**
- 알바 / 매니저 / 점주 포함
- 보통 1개 매장이지만, **2개 이상 매장에 소속될 가능성** 존재

### Secondary User (권한자)
- 점주 / 매니저
- 스케줄 생성·수정 권한 보유

---

## 3. 핵심 문제 정의 (Problem)

1. 스케줄이 매장별로 흩어져 있음
2. 오늘 내가 어디서 언제 일하는지 한 눈에 안 보임
3. 공지, 발주, 이벤트가 카톡/종이/기억에 의존
4. 스케줄 자동 생성은 필요하지만 **완전 자동은 불신**

→ **개인 관점 통합 + 매장 단위 상세**가 동시에 필요

---

## 4. 서비스 구조 핵심 원칙

1. **Home = 개인 통합 관점**
2. **매장 = 하나의 그룹 단위 (소모임 구조)**
3. 공지는 매장에 종속 (Notices 단독 메뉴 ❌)
4. 색상과 시간 블록으로 먼저 인지
5. 자동 생성 + 수동 미세 조정 공존

---

## 5. 전체 정보 구조 (IA)

```
[Login]
  └─ [Home]
        ├─ Calendar
        ├─ Store (매장 그룹)
        │     ├─ Store Home
        │     ├─ Schedule
        │     ├─ Notices
        │     ├─ 폐기 임박 목록
        │     └─ People
        └─ My Page
```

- 모바일: 하단 탭 (Home / Calendar / + / Store / My)
- 웹: 좌측 사이드바

---

## 6. Home (개인 통합 대시보드)

### 6-1. 목적
- 앱을 켜는 순간 **오늘의 나**가 보이게
- 여러 매장이어도 한 화면에서 통합

### 6-2. 주요 섹션

1. **지금 근무 상태 카드**
   - 근무 중 / 예정 / 휴무
   - 매장명 + 매장 색상 태그

2. **다음 근무**
   - 가장 가까운 다음 일정 1건

3. **My TODO (개인)**
   - 매장 무관
   - 빠른 체크용

4. **오늘의 Notices 요약**
   - 각 매장별 중요 공지
   - 오늘 새로 추가된 공지

### 6-3. 알림 구조
- Home 내부 알림함 ❌
- **우측 상단 알림 아이콘**
- 드롭다운 형태 (앱 UX)

---

## 7. Calendar (핵심 사용 화면)

### 7-1. 상단 고정 컨트롤

- [일] [주] [월] View Toggle
- 기본 필터는 최소화

### 7-2. 상세 필터 (별도 영역)

- 기간 선택 (좌우 스크롤 가능)
- 매장 (멀티 선택)
- 사람 (동료 포함)

### 7-3. 캘린더 표현 규칙

- 색상 = 사람
- 이름은 블록 시작점에만 표시
- 연장 블록은 색만 표시
- 2명 이상 겹치면 블록 분리

### 7-4. 이벤트 표시

- 회식 / 임시 휴업 / 전체 청소 등
- 스케줄과 다른 톤

### 7-5. 기타 기능
- 이미지로 다운로드 가능 (공유/출력 목적)

---

## 8. Store (매장 그룹 구조)

> 매장은 하나의 **소모임(Group)** 단위로 동작
> Home/Calendar는 개인 중심, **Store는 운영 중심**

---

### 8-1. Store 진입 구조

```
[Store 리스트]
 ├─ YOOPS 강남점
 ├─ YOOPS 홍대점
 └─ + 매장 추가 / 참여

→ 매장 선택 시, Store 내부로 진입
```

- 매장 색상 고정 (Home / Calendar 공통 사용)
- 매장 전환은 상단 Dropdown 또는 Swipe

---

### 8-2. Store 내부 IA (상단 탭)

```
[Store Home] | [Schedule] | [Notices] | [폐기 임박] | [People]
```

- 모바일: 상단 탭 스와이프
- 웹: 서브 네비게이션
- **Store Settings는 탭에 노출 ❌ (Owner만 접근)**

---

### 8-3. Store Home (운영 요약 대시보드)

#### 목적
- 매장에 들어오자마자 **오늘 매장 상태 파악**

#### 구성

1. 매장 기본 정보 카드
   - 매장명 / 색상
   - 오늘 운영 시간
   - 오늘 휴무 여부

2. 오늘 근무자 요약
   - 시간순 리스트
   - 색상 = 사람

3. 다가오는 이벤트
   - 회식 / 청소 / 임시 휴업

4. 중요 공지 Top 3
   - 중요 플래그 기준

---

### 8-4. Schedule (매장 스케줄 관리)

> **생성은 여기서, 조회는 Calendar에서**

#### 권한별 액션
- Staff: 조회 + 수정 요청
- Manager / Owner: 생성 / 수정

#### 주요 기능

1. 스케줄 생성 (기간 단위)
   - Rule 기반 자동 생성
   - 생성 전 미리보기 필수

2. 스케줄 수정
   - 드래그 & 드롭
   - 다중 인원 근무 가능

3. 변경 이력
   - 누가 언제 수정했는지 표시

---

### 8-5. Notices (매장 공지)

#### 구조

```
[전체] [중요] [발주] [금지] [기타]
```

- 하나의 리스트 + 타입 필터
- 댓글 없음
- 수정 시 이력 남김

#### 권한
- Staff: 읽기 / 북마크
- Manager / Owner: 작성 / 수정

---

### 8-6. 폐기 임박 목록

> 공지와 **완전히 분리된 실무 영역**

- 품목명
- 폐기 D-day
- 상태 체크

※ v1에서는 알림 없음

---

### 8-7. People (구성원)

#### 구성
- Owner (표시 고정)
- Manager
- Staff

#### 기능
- 역할 표시
- Manager 권한 부여 / 회수 (Owner만)
- 초대 상태 관리

---

### 8-8. Store Settings (Owner 전용)

> 메뉴에서 직접 노출 ❌
> People → 설정 진입

- 매장 정보 수정
- 운영 정책
- 매장 색상
- 초대 코드 관리

---

## 9. Notices (매장 공지)

### 9-1. 특징
- 댓글 없음
- 읽음 / 북마크

### 9-2. 타입 혼합
- 발주 목록
- 발주 금지 목록
- 기타 공지

### 9-3. 폐기 임박 목록 (분리)
- Notices와 분리된 메뉴
- 추후 알림 연계 예정
