# 매장 스케줄 관리 SaaS 기획서 (Product Spec)

## 1. 프로젝트 개요

매장 운영을 위한 **직원 근무 스케줄 관리 SaaS**.

핵심 목표 - 매장 스케줄 자동 생성 - 모바일 중심 일정 확인 - 템플릿 기반
반복 스케줄 생성 - 휴무 / 퇴사 관리

주 사용자는 **매장 점장 및 직원**.

---

# 2. 핵심 개념

## 2.1 스케줄 구조

스케줄은 두 단계 구조로 운영된다.

1.  Template (스케줄 패턴)
2.  Schedule (실제 생성된 스케줄)

Template은 패턴이며 실제 스케줄은 별도의 데이터로 생성된다.

Template과 Schedule은 **직접적인 FK 관계를 가지지 않는다.**

---

## 2.2 템플릿 기반 스케줄 생성

사용자는 **1주일 템플릿을 먼저 만든다.**

예

월\
A 09:00 \~ 18:00

화\
B 09:00 \~ 18:00

이 템플릿을 기반으로 **기간을 지정하여 스케줄을 자동 생성한다.**

예

"2026-03-01 \~ 2026-03-31 자동 생성"

---

## 2.3 자동 생성 규칙

자동 생성 시 동작

1.  기존 스케줄 확인
2.  스케줄 존재 시
    - 충돌 날짜 표시
    - 경고 알림 표시
3.  진행 시
    - 해당 기간 스케줄 전체 삭제
    - 새 스케줄 생성

---

# 3. 휴무 처리

휴무는 직원 기준으로 설정된다.

자동 생성 시 규칙

휴무일이면 해당 직원 스케줄 생성하지 않음

휴무 우선 원칙 적용

---

# 4. 퇴사 처리

직원이 퇴사하면

1.  employment.resign_date 설정
2.  오늘 이후 미래 스케줄 삭제
3.  과거 스케줄 유지

또한

- 템플릿에서 제외
- 새 스케줄 생성 대상에서 제외

삭제 방식

Soft Delete

---

# 5. 스케줄 수정 정책

스케줄 생성 이후

- 관리자 수동 수정 가능
- 수정 즉시 반영

Draft 상태 없이 바로 확정 스케줄로 저장된다.

---

# 6. 히스토리 기록

모든 스케줄 변경은 히스토리를 기록한다.

예

A 09:00\~18:00\
→\
A 10:00\~18:00

MVP에서는 **스케줄 변경만 기록한다.**
