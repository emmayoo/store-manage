# 기획서·DB 스키마 검토 피드백

프로젝트 구성 시 발견한 사항과 수정/권장 사항을 정리했습니다.

---

## 1. DB 스키마 문서 수정 반영

### 1.1 `schedule_histories` 테이블 오타

- **문서**: ` to_be` (공백 포함)
- **조치**: 컬럼명을 `to_be`로 통일해 Prisma 스키마 및 마이그레이션에 반영했습니다.

### 1.2 `schedule_histories` 역관계

- **내용**: `User` 모델에 `scheduleHistories` 관계를 두려면 `ScheduleHistory`에 `changedBy`에 대한 User 역관계가 필요합니다.
- **조치**: `ScheduleHistory.changedByUser` → `User` 관계를 추가했습니다.

## 2. 기획서 관련 권장 사항

### 2.1 MVP 범위와 운영 메뉴

- **기획서 9.3**: 운영 메뉴에 **상품 관리**, **공지 관리**가 포함되어 있으나, **11. MVP 범위**에는 포함되지 않습니다.
- **권장**: MVP에서는 해당 메뉴를 “준비 중” 등으로 두고, 실제 기능은 제외하거나 추후 단계에서 구현하는 것이 일관적입니다. 현재는 플레이스홀더 페이지만 두었습니다.

### 2.2 스케줄 자동 생성 시 “충돌” 처리

- **기획서 2.3**: “스케줄 존재 시 충돌 날짜 표시, 경고 알림 표시” → **11. MVP 범위**에서는 “스케줄 충돌 자동 검증”이 **제외**입니다.
- **현재 구현**: 자동 생성 시 해당 기간 스케줄을 **전체 soft delete 후 재생성**하는 방식으로만 구현했습니다. 충돌 경고/선택 생성은 MVP 이후로 두는 것이 좋습니다.

### 2.3 인증(로그인) 플로우

- **기획서**: Auth - JWT (httpOnly Cookie)로 명시되어 있으나, 로그인/회원가입 API·화면은 별도 정의가 없습니다.
- **현재**: `lib/auth.ts`에 토큰 발급/검증/쿠키 설정만 구현했습니다. 로그인 폼 및 회원가입은 추후 연동이 필요합니다.

### 2.4 캘린더 “겹치는 스케줄” UI

- **기획서 7.5**: 겹치면 좌우 분할 표시, 겹침 허용.
- **현재**: Day View는 리스트 형태로만 구현했습니다. 시간축 기반 레이아웃에서 겹침 시 좌우 분할 표시는 프론트 전용 작업으로 추후 구현하는 것이 좋습니다.

### 2.5 퇴사 처리 자동화

- **기획서 4**: 퇴사 시 employment.resign_date 설정, 오늘 이후 미래 스케줄 삭제, 템플릿에서 제외 등.
- **현재**: 자동 생성 시 퇴사자(resign_date 설정된 사용자)는 **생성 대상에서 제외**했습니다. “오늘 이후 미래 스케줄 삭제”는 별도 API(예: 퇴사 처리 시 호출)로 두는 것을 권장합니다.

## 3. 기술 스택·버전 정리

- **Next.js**: 16.x (React 19)
- **Prisma**: 7.x (Driver Adapter 사용, `@prisma/adapter-pg` + `pg`)
- **DB**: Supabase PostgreSQL (연결 문자열만 설정하면 됨)
- **UI**: Tailwind v4, shadcn/ui (canary, Tailwind v4 대응)
- **PWA**: `app/manifest.ts`로 manifest 제공 (Service Worker는 필요 시 추후 추가)

---

## 4. 환경 변수

- `.env.example` 참고.
- **필수**: `DATABASE_URL`, `JWT_SECRET`
- **선택**: Supabase URL/Anon Key (Storage 등 사용 시)

이 문서는 기획·스키마 변경 시 함께 업데이트하는 것을 권장합니다.
